# x402 HTTP payments

[x402](https://github.com/coinbase/x402) is a pattern where a resource responds with **HTTP 402** and machine-readable **payment requirements**; the client pays on-chain (here: **Algorand USDC** per the [exact Algo scheme](https://github.com/coinbase/x402/blob/main/specs/schemes/exact/scheme_exact_algo.md)), then retries the request with proof of payment.

## Using x402 with AlgoPay

Pass an **`https://` URL** as `recipient` to **`pay()`**:

```python
result = await client.pay(
    wallet_id,
    "https://api.example.com/protected/resource",
    "2.5",  # max USDC you allow this resource to charge
)
if result.success and result.resource_data is not None:
    # Final response body / parsed payload depends on adapter
    print(result.resource_data)
```

The **`X402Adapter`** (see `algopay.protocols.x402`) handles discovery, quote acceptance within your cap, signing, and the paid retry.

## Dependencies

The package depends on **`x402-avm[avm]`** and **`httpx`** for HTTP and protocol handling (see `pyproject.toml` on [PyPI](https://pypi.org/project/algopay-sdk/0.1.0a1/)).

## Environment and examples

Example script in the repo: **`examples/x402_client_demo.py`**.

| Variable | Purpose |
| -------- | ------- |
| `ALGOPAY_X402_URL` | HTTPS URL that returns x402 payment requirements |
| `ALGOPAY_MAX_USDC` | Maximum USDC for the demo client |
